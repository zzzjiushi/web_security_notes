# 一、什么是文件包含（你要在脑子里形成的模型）

一句话定义：

> **程序把“用户可控的字符串，当作文件路径去加载/执行”。**

在 PHP 中最常见的就是：

```
include($_GET['page']);
require($_GET['file']);
include_once($a);
```

你需要形成的思维是：

> **“我输入的不是参数，是服务器要去打开的文件路径。”**

------

# 二、如何一眼识别“这是文件包含题”

看到下面任意一个特征，你就该警觉：

### 1️⃣ 参数名高度可疑

```
page=
file=
path=
template=
lang=
view=
```

### 2️⃣ 页面行为异常

- 页面结构不变，但内容在变
- 报错类似：
  - `failed to open stream`
  - `No such file or directory`
  - `include(): Failed opening`

### 3️⃣ 源码中出现（最明确）

```
include $_GET['page'];
require $file;
```

**比赛中：源码 > 报错 > 行为变化**

------

# 三、文件包含的两大核心类型（非常重要）

## 1️⃣ LFI（本地文件包含）

> 包含的是**服务器本地已有的文件**

### 典型目标

- 配置文件
- 源码
- 日志文件
- 临时文件

你在脑中要想的是：

> “我能不能让它去读一个**本来不该让我看的文件**？”

------

## 2️⃣ RFI（远程文件包含）

> 包含的是**远程 URL 上的文件**

```
include($_GET['page']); // page=http://xxx/shell.txt
```

⚠️ **现在比赛中较少**，通常需要：

- `allow_url_include = On`

------

# 四、LFI 的核心利用思路（不是 payload）

你在比赛中不要想“我用哪个字符串”，而要想**这 4 条路能不能走通**：

------

## 思路 1：目录穿越 → 读系统文件 / 配置

### 思维模型

> 我能不能跳出当前目录，读到别的地方？

你要测试的是：

- 是否能向上跳目录
- 是否有路径过滤

成功标志：

- 页面内容变化
- 报错内容变化

------

## 思路 2：读 PHP 源码（CTF 极高频）

### 关键问题

> **PHP 文件默认会被执行，那我怎么“只读不执行”？**

这时候你脑子里就该出现：

- **协议封装器**
- **编码读取**

目标只有一个：

> **拿到源码，看 flag 在哪 / 看下一步漏洞**

------

## 思路 3：日志 / 临时文件 → 间接执行

这是偏进阶，但**CTF 很爱考**。

### 思维模型

1. 服务器有没有日志文件？
2. 日志内容里，能不能被我控制？
3. 我能不能把“我控制的内容”再 include 进来？

你不需要现场想全流程，只要记住一句话：

> **“可控内容 + include = 有机会代码执行”**

------

## 思路 4：配合其他漏洞（文件包含很少单独结束）

文件包含**本身不一定直接给 flag**，它经常是中转站：

- 文件包含 + 上传 → 执行上传文件
- 文件包含 + 反序列化 → 触发魔术方法
- 文件包含 + 配置文件 → 拿数据库密码

------

# 五、比赛中“做文件包含”的标准流程（非常实用）

这是你明天**真正该照着走的流程**：

------

### Step 1：确认是不是文件包含

- 参数名
- 报错
- 行为变化

❌ 不确认就乱试是浪费时间

------

### Step 2：判断是 LFI 还是 RFI

- 本地路径是否有效？
- 能否加载 URL？

------

### Step 3：优先目标 = 源码

问自己一句话：

> **“出题人最希望我看到哪个 PHP 文件？”**

90% 是：

- `index.php`
- 当前页面自身
- 配置文件

------

### Step 4：看源码再决定下一步

- 有 flag → 结束
- 有新参数 → 继续挖
- 有过滤 → 思考绕过点

------

## 六、你现在必须记住的 5 句话（赛场版）

1. **文件包含 = 用户输入 → 文件路径**
2. **LFI 先想读源码，而不是急着 RCE**
3. **源码读出来，题目往往就“摊牌了”**
4. **include 本身不是终点，是跳板**
5. **看报错信息，比乱试字符串重要**
