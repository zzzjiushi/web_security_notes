# 关于伪造请求头

## 一句话结论（你赛场上该怎么用）

> **当你怀疑“后端在信任某些请求头”时，就该立刻尝试伪造请求头。**

不是每个题都用，但**一旦用对，往往是秒解点**。

------

## 一、什么时候你该“条件反射”想到伪造请求头？

下面这些**任意出现一个**，你就该想到“要不要加请求头”。

------

### 1️⃣ 题目或页面出现“身份 / 来源 / 内部”的暗示

例如：

- 提示文字里有：
  - `internal`
  - `admin`
  - `only localhost`
  - `only internal access`
- 报错信息类似：
  - `only allow local access`
  - `your ip is not allowed`

👉 **第一反应：它可能在判断来源 IP 或身份**

------

### 2️⃣ 功能“明明存在，但你访问不到”

典型场景：

- `/admin` 存在，但 403
- `/debug` / `/api` 返回 “permission denied”
- 某个功能在前端隐藏，但后端接口还在

👉 **说明：后端可能不是靠登录态，而是靠请求头判断**

------

### 3️⃣ 源码里出现这些东西（这是最明确的）

如果你能看到源码，重点盯：

```
$_SERVER['HTTP_X_FORWARDED_FOR']
$_SERVER['HTTP_CLIENT_IP']
$_SERVER['REMOTE_ADDR']
$_SERVER['HTTP_REFERER']
$_SERVER['HTTP_USER_AGENT']
```

或者类似逻辑：

```
if ($ip === '127.0.0.1') { ... }
```

👉 **只要看到 `$_SERVER`，就要想“这个值我能不能伪造？”**

------

## 二、最常见的 6 种“比赛级请求头伪造点”

你不需要记很多，就这几个，**覆盖 90% Web 题**。

------

### 1️⃣ `X-Forwarded-For`（最重要）

#### 用途（出题人常用）

- 判断是否本地访问
- 判断是否管理员 IP

#### 你在想什么

> “如果后端用它当真实 IP，那我就能假装是内网 / 本地”

#### 典型值

```
X-Forwarded-For: 127.0.0.1
```

------

### 2️⃣ `X-Real-IP`

和上面类似，有些框架用这个。

```
X-Real-IP: 127.0.0.1
```

------

### 3️⃣ `Referer`

#### 用途

- 判断是否从“合法页面”跳转
- 防 CSRF 的弱实现

```
Referer: http://localhost/admin
```

👉 比赛中经常是**摆设型校验**

------

### 4️⃣ `User-Agent`

#### 用途

- 判断是不是爬虫
- 判断是不是管理员工具

```
User-Agent: admin
```

有些题甚至写：

```
if (strpos($_SERVER['HTTP_USER_AGENT'], 'admin') !== false)
```

------

### 5️⃣ 自定义 Header（最“CTF 味”的）

例如：

```
if ($_SERVER['HTTP_TOKEN'] === 'admin') { ... }
```

你就直接加：

```
Token: admin
```

👉 **看到 `HTTP_XXX`，就等于告诉你“加个 XXX 头试试”**

------

### 6️⃣ Host 头（少见但要记）

```
Host: localhost
```

在一些 SSRF / 内网逻辑判断中会用到。

------

## 三、你在比赛中“伪造请求头”的标准动作

这是你**真正该形成的操作习惯**。

------

### Step 1：抓包（Burp）

你不是靠浏览器地址栏，而是**一定进 Burp Repeater**。

------

### Step 2：只加，不删（新手很重要）

先不要删原有头，只在下面**加一行**：

```
X-Forwarded-For: 127.0.0.1
```

看响应是否变化。

------

### Step 3：观察“变化”，不是一次成败

关注：

- 状态码是否变了
- 返回内容是否多了
- 报错信息是否变了

**变化 ≫ 成功**

------

### Step 4：多头组合（CTF 常见）

例如：

```
X-Forwarded-For: 127.0.0.1
X-Real-IP: 127.0.0.1
Referer: http://localhost/
```

出题人经常写得很“随意”。

------

## 四、你要避免的两个误区

### ❌ 误区 1：没提示就不敢试

在 CTF 中：

> **请求头 = 成本极低的尝试**

加一行 header 几乎不花时间。

------

### ❌ 误区 2：以为这是“高级技巧”

恰恰相反：

> **请求头伪造是“出题人送分用的低成本信任错误”**

------

## 五、给你一条“赛场触发条件口诀”

你可以直接背这一句：

> **“权限不够 / 内部接口 / IP 校验 / 源码用 $_SERVER —— 先加 X-Forwarded-For。”**
