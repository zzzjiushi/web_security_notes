# Path traversal

路径遍历一共分为三部分：

- 什么是路径遍历
- 如何实施路径遍历并规避常见障碍
- 如何防范路径遍历漏洞

## 什么是路径遍历

路径遍历也称目录遍历。此类漏洞使攻击者能够读取运行应用程序的服务器上的任意文件：

- 应用程序代码和数据
- 后端系统的凭证
- 敏感的操作系统文件

在某些情况下，攻击者可能能够向服务器上的任意文件写入内容，从而修改应用程序数据或行为，最终完全控制服务器。



## 如何实施路径遍历并规避常见障碍

### 通过路径遍历读取任意文件

举例：

一个购物应用程序，它会显示待售商品的图片。该应用程序可能使用以下HTML加载图片：

```html
<img src="/loadImage?filename=1.png">
```

该 `loadImage` URL接受一个filename的参数，并返回指定文件的内容。图像文件存储在磁盘的以下位置：`/var/www/images` :要返回图像，应用程序会将请求的文件名附加到此基础目录，并使用文件系统API读取文件内容。也就是说：应用程序从以下路径读取内容：

```
/var/www/images/1.png
```

如果应用程序未实施任何防御措施来抵御路径遍历攻击，攻击者可通过请求以下URL从服务器文件系统中获取文件 `/etc/passwd` :

```
http://xxx.com/loadImage?filename=../../../etc/passwd
```

这样做的结果就是：导致应用程序从以下文件路径中读取：

```
/var/www/images/../../../etc/passwd
```

该序列在文件路径 `../` 中有效，表示向上易懂目录结构一级。连续三个序列从当前目录 `/var/www/images/` 逐步移到文件系统的跟目录。

在基于Unix的操作系统上，这是包含服务器注册用户详细信息的标准文件，但攻击者利用相同技术获取其他任意文件。

在Windows系统中，和 `../`和 `..\` 都是有效的遍历序列。



### 利用路径遍历漏洞的常见障碍

许多将用户输入写入文件路径的应用程序都实现了防范路径遍历攻击的机制，但这些机制往往可以被绕过。

如果应用程序从用户提高的文件名中剥离或阻止目录遍历序列 `(../)`,则可能通过多种绕过：

#### 绝对路径绕过

可以直接使用文件系统根目录下的绝对路径（filename=/etc/passwd），直接引用文件而无需任何路径遍历序列。

#### 嵌套遍历序列

可以使用嵌套遍历序列，例如：

```
....//或....\/
```

当内部序列被剥离时，这些序列会还原为简单的遍历序列：

```
输入：
..../
拆解：
..+..//
如果程序先匹配../
删除内部 ../
就会剩下：
..//
多个/会被系统折叠成为一个/
```

这就是利用解析规则与字符串替换规则不一致



#### URL编码绕过

在某些场景下，例如URL路径或 `multipart/form-data` 请求中的filename参数里，web服务器可能会把你的输入传递给应用程序之前，先移除所有的目录遍历序列。

有时候可以通过对 `../` 进行URL编码，甚至双重URL编码来绕过这种过滤机制例如：

```
../  —— %2e%2e%2f
双重编码：%252e%252e%252f
```

此外，一些非标准编码形式也可能生效，例如：

```
..%c0%af
..%ef%bc%8f
```



#### 预期文件名开头绕过

应用程序可能要求用户提供的文件名以预期基础文件夹开头，例如 `/var/www/images` 。在这种情况下，可以包含所需的基础文件夹，后面跟随适当的遍历序列。

```
filename=/var/www/images/../../../etc/passwd
```



#### 预期文件扩展名绕过(空字节截断)

应用程序可能要求用户提供的文件名以预期的文件扩展名结尾，例如 `.png` 。在这种情况下，可能通过使用空字节在所需扩展名前有效终止文件路径。例如：

```
filename=../../../etc/passwd%00.png
```



## 如何防范路径遍历攻击

防止路径遍历漏洞最有效的方法是完全避免将用户提供的输入传递给文件系统API。许多执行此操作的应用程序都可以重写，以更安全的方式实现相同的行为。

若无法避免将用户的输入传递给文件系统API,我们建议采用双重防御机制来防范攻击：

- 在处理用户输入前进行验证。理想情况下，应将用户输入与允许值的白名单进行对比。若无法实现，则需验证输入内容仅包含允许的字符，例如仅限字母数字字符。
- 在验证提供的输入后，将输入追加到基目录，并使用平台文件系统API规范化路径。验证规范化路径是否以预期的基目录开头。

以下是一个简单的Java代码示例，用于根据用户输入验证文件的规范路径：

```
File file = new File(BASE_DIRECTORY, userInput); if (file.getCanonicalPath().startsWith(BASE_DIRECTORY)) {    // process file }
```
